## Назначение

CS-Styles — CSS-библиотека для семантической HTML-разметки. Ее задача — дать готовую базу для интерфейсов, где основной упор сделан на HTML5 и ARIA, а классы используются только там, где это действительно нужно.

Библиотека покрывает:

- базовую типографику и сбросы
- цветовые темы и токены
- компоненты форм, кнопок и навигационных элементов
- утилиты для компоновки и размеров
- vendor-слой для совместимых вспомогательных стилей

**[Демо](https://si9urd.github.io/cs-styles-demo/)**

## Установка

```bash
npm i @oleh-ob/cs-styles
```

## Подключение

Основной файл стилей:

```css
@import "@oleh-ob/cs-styles/dist/cs-styles.min.css";
```

Если нужен исходный CSS без минификации, используйте `dist/cs-styles.css`.

## Публичный API

Публичный интерфейс библиотеки состоит из следующих точек входа и контрактов:

- `dist/cs-styles.min.css` — основной production-артефакт
- `dist/cs-styles.css` — читаемая не минифицированная сборка
- `scss/cs-styles.scss` — исходная SCSS-точка входа для сборки в собственных пайплайнах
- CSS-переменные вида `--cs-*` — токены темы, размеров и состояния
- `data-theme="dark"` — публичный механизм переключения темы
- семантические элементы HTML и ARIA-атрибуты — основной способ активации стилей

## Что входит

- семантическая базовая стилизация
- темы и цветовые схемы
- компоненты форм и кнопок
- утилиты для spacing, sizing, display, borders, align и text
- vendor-слой для совместимости и вспомогательных примесей

## Сборка

```bash
npm run build
```

Доступные скрипты:

- `npm run build:css` — сборка SCSS в CSS
- `npm run build:autoprefix` — добавление префиксов через PostCSS
- `npm run build:minify` — минификация CSS
- `npm run watch` — пересборка при изменении `scss/`
- `npm run lint` — запуск проверок форматирования и сортировки
- `npm run theme` — генерация цветовой темы из JSON-источника

## Структура

- `scss/` — исходники библиотеки
- `dist/` — собранные CSS-артефакты
- `material-theme*.json` — исходные данные для генерации тем
- `material-theme.js` — генератор темы

## Поддержка браузеров

Целевая поддержка определяется `browserslist` в `package.json` и ориентируется на современные браузеры. Internet Explorer не поддерживается.
